### Uncontrolled vs Controlled State

To use the date field in the uncontrolled state, set the `defaultValue` prop to the initial value of the date field.

To use the date field in the controlled state, set the `value` prop to the current value of the date field and provide an `onValueChange` event listener to handle the change.

<Tabs items={["Preview", "Code"]} defaultValue="Preview">
  <Tab value="Preview">
    <div className="not-prose flex flex-col gap-3 items-center">
        <div>
            Uncontrolled
            <DateField defaultValue={NOW} />
        </div>
        <div>
            Controlled
            <DateField value={NOW} />
        </div>
    </div>
  </Tab>
  <Tab value="Code">
    ```tsx
    // Uncontrolled state
    <DateField defaultValue={new Date().toISOString().split("T")[0]} />

    // Controlled state
    <DateField value={new Date().toISOString().split("T")[0]} />
    ```

  </Tab>
</Tabs>

### Disabled

Date field can be disabled by setting the `disabled` prop to `true`. When disabled, the date field will not be interactive.

<Tabs items={["Preview", "Code"]} defaultValue="Preview">
  <Tab value="Preview">
    <div className="not-prose flex justify-center">
      <DateField defaultValue={NOW} disabled />
    </div>
  </Tab>
  <Tab value="Code">
    ```tsx
    <DateField defaultValue={new Date().toISOString().split("T")[0]} disabled />
    ```
  </Tab>
</Tabs>

### Invalid

Invalid state is triggered when a date field fails validation. To indicate an invalid date field, set the `invalid` prop to `true`.

<Tabs items={["Preview", "Code"]} defaultValue="Preview">
  <Tab value="Preview">
    <div className="not-prose flex justify-center">
      <DateField defaultValue={NOW} invalid />
    </div>
  </Tab>
  <Tab value="Code">
    ```tsx
    <DateField defaultValue={new Date().toISOString().split("T")[0]} invalid />
    ```
  </Tab>
</Tabs>
